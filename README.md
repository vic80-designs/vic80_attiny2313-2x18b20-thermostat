# Двухканальный термостат на ATtiny2313 с энкодером (asm)

Универсальный двухканальный термостат с управлением инкрементальным энкодером, написанный на чистом модульном ассемблере AVR. Проект разработан более 10 лет назад и успешно проверен длительной реальной эксплуатацией.

Предназначен для автоматического управления климатом и защиты отопительных систем или электроприборов от перегрева. Подходит как для водяных бойлеров (контроль жидкого теплоносителя), так и для электрических конвекторов (контроль сухого ТЭНа).

## 🚀 Ключевые особенности
* **Аппаратный минимализм:** Всё реализовано на одной ATtiny2313. Опрос датчиков, обработка ввода и динамическая индикация работают напрямую, без сдвиговых регистров и дешифраторов.
* **Логика работы «И / ИЛИ»:** Включение нагрева происходит только по условию логического **И** (когда остыли оба датчика), а выключение — по логическому **ИЛИ** (как только один из датчиков достиг предела).
* **Система самодиагностики (Ошибки):** При обрыве или неисправности датчиков нагрузка мгновенно обесточивается, а на экран выводится аварийный код: `Er1` (сбой Д1), `Er2` (сбой Д2) или `Er3` (сбой обоих датчиков).
* **Конфигурация дисплея:** Код адаптируется под индикаторы как с Общим Анодом (ОА), так и с Общим Катодом (ОК) изменением всего одного дефайна в начале исходного кода.
* **Эстетичный интерфейс:** Поддерживается индикация отрицательных температур с динамическим сдвигом знака «минус» вплотную к числу (вывод ` -1` или `-10` без отрыва знака от цифры).

## 📊 Алгоритм контроля температуры
1. **Включение нагрузки (Логическое И):** Датчик 1 (Теплоноситель/ТЭН) < (Уставка 1 - Гистерезис 1) **И** Датчик 2 (Воздух) < (Уставка 2 - Гистерезис 2).
2. **Выключение нагрузки (Логическое ИЛИ):** Датчик 1 >= Уставка 1 **ИЛИ** Датчик 2 >= Уставка 2.

## 📁 Структура репозитория
* `📁 Firmware/Bin — готовые скомпилированные файлы прошивок отдельно для индикаторов с Общим Анодом (Thermostat_CA.hex) и Общим Катодом (Thermostat_CC.hex).
* `📁 Firmware/Source_Code` — полноценное модульное решение для Microchip Studio (Atmel Studio) со всеми исходниками на ассемблере, разбитыми по функциональным узлам (`/DynIndik`, `/Encoder`, `/OWI` и т.д.).
* `📁 Schematics` — принципиальная схема устройства, представленная в форматах **PDF** (для качественной печати) и **PNG** (для быстрого просмотра в браузере). По умолчанию схема сконфигурирована под Общий Катод (ОК).
* `📁 PCB` — Файлы печатной платы: Gerber-архив в формате .zip для заказа плат на заводе и изображения слоев.

## 🛠 Инженерное меню (Параметры)
Вход осуществляется удержанием кнопки энкодера более 3 секунд.
* **П-1:** Верхнее ограничение для уставки датчика 2 (Воздух). *Диапазон: 0...45 °C*.
* **П-2:** Гистерезис датчика 2 (Воздух). *Диапазон: 1...10 °C*.
* **П-3:** Уставка температуры для датчика 1 (Теплоноситель/ТЭН). *Диапазон: 10...127 °C*.
* **П-4:** Гистерезис датчика 1 (Теплоноситель/ТЭН). *Диапазон: 1...20 °C*.
